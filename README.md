# EVzone Rider App

A React 18 + TypeScript + Vite Progressive-Web-App-style shell for the EVzone Rider / Customer app.

## Tech stack

- React 18.2
- TypeScript 5.x
- Vite 7
- react-router-dom v7
- MUI v5 + Tailwind CSS 3
- `@react-google-maps/api` for maps
- `socket.io-client` for realtime updates

## Quick start

```bash
npm install
npm run dev
```

Then open http://localhost:5173 in your browser.

## Environment variables

Copy `.env.example` to `.env.local` and configure:

```bash
VITE_BACKEND_BASE_URL=http://localhost:3000/api/v1
VITE_SOCKET_BASE_URL=http://localhost:3000
VITE_GOOGLE_MAPS_API_KEY=...
VITE_BACKEND_ENABLED=true
```

## Backend integration

The app communicates with the EVzone backend (`/home/developer/Projects/backend`):

- REST API under `/api/v1`
- Socket.io namespace `/rider`
- JWT stored in `localStorage`

Key ride flow screens:
- `/rides/enter/details` – pickup / destination input
- `/rides/options` – vehicle class & fare estimate
- `/rides/searching` – searching for a driver
- `/rides/driver-on-way` – driver assigned / en route
- `/rides/driver-arrived` – OTP verification
- `/rides/trip` – in-progress trip

## Scripts

- `npm run dev` – start Vite dev server
- `npm run build` – production build
- `npm run typecheck` – run TypeScript without emit

## Notes

- The app no longer uses Create React App or `.jsx` files.
- Do not commit `dist/` or `build/` by hand; they are regenerated by the build.
